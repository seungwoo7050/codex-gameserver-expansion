# v0.1.0 서버 설계 요약

## 목표
- 단일 io_context 기반 C++17 서버 스켈레톤 제공
- HTTP `/api/health`와 WebSocket 에코 `/ws` 구현

## 구조
- Boost.Beast HTTP 서버: `HttpSession`이 요청 파싱 및 엔드포인트 분기
- WebSocket: 업그레이드 요청을 받아 `WebSocketSession`에서 에코 처리
- 설정 로딩: 환경변수를 `AppConfig`로 수집
- 실행 모델: io_context + 워커 스레드 풀, 서버 시작/정지는 `ServerApp`에서 관리

## 동시성/비차단 정책
- io_context는 워커 스레드로 병렬 처리
- DB/Redis는 이 버전에서 연결만 고려하며 I/O 스레드에서 블로킹 호출 없음

## 테스트
- 단위: JSON 응답 엔벨로프 검증
- E2E: `/api/health` HTTP 호출, WebSocket 에코 왕복
