# v0.7.0 관측성 정의

## 로그 필드
- traceId: 요청/이벤트 추적 식별자(서버가 생성)
- eventName: HTTP 경로 또는 WS 이벤트명
- latencyMs: 처리 지연(ms)
- userId: 사용자 식별자(가능 시)
- sessionId: 세션 식별자(가능 시)

## 메트릭 필드
- requests.total: 누적 HTTP 요청 수
- requests.errors: 누적 HTTP 오류 수(상태코드 4xx/5xx 기준)
- connections.websocket: 활성 WebSocket 연결 수
- sessions.active: 활성 게임 세션 수
- queue.length: 매칭 큐 대기열 길이

## 민감정보 차단 규칙
- 토큰, 비밀번호, 개인 식별 문자열은 로그/메트릭에 포함하지 않는다.
- /metrics와 /ops/status 응답은 숫자/상태 값만 노출한다.

## 운영 보호 정책
- /ops/status: 헤더 `X-Ops-Token`이 `${OPS_TOKEN}`과 일치해야 접근 가능.
- /metrics: 민감정보가 없으므로 토큰 없이 읽기 가능.
